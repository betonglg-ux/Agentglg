# GitHub Mirror Manifest

Репозиторий: `betonglg-ux/Agentglg`

Назначение: этот манифест описывает стартовую структуру GitHub-зеркала для текущего агента. Текущий агент продолжает работать со своими собственными инструкциями, файлами, навыками и Memory, а GitHub хранит их дубли, историю развития и материалы для быстрого создания будущих похожих агентов.

## Базовые принципы
- текущая конфигурация агента является основной рабочей средой;
- GitHub является внешним зеркалом и архивом развития;
- важные наработки не должны оставаться только внутри Memory, если они нужны для переноса в другого агента;
- при расхождениях сначала нужно ориентироваться на актуальное состояние текущего агента, затем обновлять зеркало в GitHub.

## Что должно храниться в разделах

### `agent-development/CHANGELOG.md`
Короткий журнал значимых изменений агента:
- новые правила проверки;
- важные исправления логики;
- изменения в структуре навыков;
- изменения в шаблонах и рабочих файлах;
- значимые экспорты из памяти.

### `agent-development/confirmed-error-patterns.md`
Подтвержденные пользователем типовые ошибки, которые агент должен проверять особенно внимательно.

### `agent-development/missed-findings-log.md`
Журнал реальных пропусков агента:
- что было пропущено;
- в каком типе протокола;
- как это распознавать в будущем.

### `agent-development/template-notes.md`
Устойчивые заметки по шаблонам:
- особенности конкретных форм;
- важные различия между похожими шаблонами;
- правила сверки PDF против Excel.

### `agent-development/user-confirmed-corrections.md`
Поправки и замечания, которые пользователь явно подтвердил как верные.

### `agent-development/current-agent-instructions.md`
Актуальная рабочая версия инструкций агента в читаемом виде для переноса в будущие копии.

### `agent-development/agent-summary.md`
Краткое описание агента:
- назначение;
- основные источники истины;
- какие навыки подключены;
- какие папки и шаблоны используются;
- что именно нужно воспроизвести при создании похожего агента.

### `agent-development/memory-exports/`
Экспорт важных данных из Memory в переносимом виде.

Минимальные файлы:
- `README.md`
- `confirmed-error-patterns-export.md`
- `missed-findings-export.md`
- `template-notes-export.md`
- `user-corrections-export.md`

### `agent-development/skills/`
Зеркало материалов навыков.

Минимально:
- `README.md`
- отдельная папка на каждый важный навык;
- для текущего агента обязательно предусмотреть папку `glavlab-protocol-review/`.

### `agent-development/files-index/`
Индекс файлов агента, чтобы можно было быстро понять, какие материалы были прикреплены и зачем они нужны.

Минимальные файлы:
- `README.md`
- `attached-files-index.md`
- `templates-index.md`

### `agent-development/protocols/`
Зеркальная структура папок с шаблонами и рабочими типами протоколов.

Для старта нужно предусмотреть папки:
- `Адгезия`
- `Анкеры`
- `Бетон`
- `Влажность`
- `Металл`
- `Момент затяжки`
- `Нагрузки`
- `Сварка`
- `Толщина ЛКП`
- `Уплотнение`

## Что нужно фиксировать в GitHub в первую очередь
1. Актуальные инструкции агента.
2. Ключевой навык `glavlab-protocol-review`.
3. Индекс прикрепленных файлов и шаблонов.
4. Подтвержденные паттерны ошибок.
5. Журнал пропусков.
6. Экспорт важной памяти.
7. Changelog изменений.

## Что пригодится для размножения агента
При создании нового похожего агента нужно в первую очередь восстановить:
- инструкции из `current-agent-instructions.md`;
- структуру навыков из папки `skills/`;
- индексы и список прикрепляемых файлов из `files-index/`;
- устойчивые знания из `confirmed-error-patterns.md`, `template-notes.md`, `user-confirmed-corrections.md` и `memory-exports/`.

## Статус этого манифеста
Это стартовый манифест структуры. Его можно расширять по мере того, как агент будет накапливать новые типы материалов для дублирования в GitHub.
