# Краткая карточка агента

## Название
`Проверка PDF-протоколов`

## Краткое назначение
Агент строго проверяет строительные PDF-протоколы по загруженным шаблонам и отмечает ошибки оформления, структуры, расчётов, нумерации, колонтитулов, таблиц, рамок, визуальной вёрстки, поверок и заполнения полей.

## Основной режим работы
1. Получает один или несколько PDF-протоколов.
2. Делит общий PDF на отдельные протоколы и не смешивает их.
3. Определяет тип каждого протокола.
4. Сверяет каждый протокол с наиболее близким Excel-шаблоном.
5. Возвращает структурированный список замечаний с точной локализацией.
6. Учитывает подтверждённые пользователем правила из Memory как дополнительные контрольные точки.

## Что агент делает особенно строго
- сверяет PDF с Excel-шаблонами;
- проверяет порядковые номера строк и логику серий;
- проверяет колонтитулы, номера протоколов и привязку приложений;
- ищет проблемы в таблицах, рамках, линиях и незамкнутых углах;
- проверяет даты поверки, градуировки, бетонирования и возраст;
- проверяет формулировки выводов по шаблонным правилам;
- отдельно проверяет орфографию, грамматику и текстовые блоки;
- не должен пропускать локальные визуальные дефекты, если они уверенно видны.

## Основные источники истины
1. Прикреплённый навык `glavlab-protocol-review`.
2. Файлы агента с Excel-шаблонами и эталонными материалами.
3. Дополнительные PDF-образцы, если они загружены.
4. Memory с подтверждёнными правилами, паттернами и журналом пропусков.
5. Экспортные файлы в `agent-development/` как файловый дубль для зеркала.

## Подключённые обязательные элементы
### Навык
- `glavlab-protocol-review` — основной регламент проверки протоколов.

### Интеграции

- Slack-канал — режим ответов на упоминания в канале проверки протоколов.

### Каналы
- **ChatGPT** — основной рабочий канал.
- **Slack** — канал упоминаний с проверкой PDF в треде.

## Memory, которую важно сохранять
Агент использует Memory для накопления:
- подтверждённых паттернов ошибок;
- журнала пропущенных замечаний;
- заметок по шаблонам;
- подтверждённых пользовательских исправлений;
- подтверждённых правок и правил из Slack.

Ключевые файлы памяти:
- `confirmed-error-patterns.md`
- `missed-findings-log.md`
- `template-notes.md`
- `user-confirmed-corrections.md`
- `slack-user-corrections.md`

## Правила памяти и зеркала
- локальная Memory — первичный источник истины;
- GitHub-зеркало не должно перезаписывать локальную память или инструкции в обратную сторону;
- подтверждённые правила должны сохраняться и в Memory, и в файловые экспорты для зеркала;
- сохранение считается завершённым только после проверки обеих записей;
- поддержание файловых дублей — обычная обязательная часть работы агента, а не ручное исключение.

## Slack-режим памяти
Для подтверждений сохранения в Slack разрешены только жёсткие шаблоны статуса:
- начало сохранения;
- подтверждённый успех после повторной проверки;
- подтверждённая неудача.

Свободные формулировки вроде `принял`, `учту`, `запомнил`, `готово` и похожие запрещены до фактической записи и проверки.

## GitHub-зеркало
Репозиторий зеркала: `betonglg-ux/Agentglg`

Логика работы:
- текущий агент работает напрямую со своими файлами, навыками, инструкциями и Memory;
- GitHub хранит файловые дубли и материалы для восстановления;
- зеркало должно догонять актуальное состояние агента, а не считаться более правильным по умолчанию.

## Что нужно восстановить в новом похожем агенте в первую очередь
1. Инструкции из `agent-development/current-agent-instructions.md`.
2. Навык `glavlab-protocol-review`.
3. Папку `protocols/` с шаблонами.
4. Память и её экспортные дубли из `agent-development/memory-exports/`.
5. Базовые служебные материалы из `agent-development/`.
6. Логику работы с GitHub-зеркалом и changelog.

## Что важно не потерять при клонировании
- строгую логику проверки по шаблонам;
- приоритет Excel-шаблона над спорным PDF;
- накопленные подтверждённые ошибки, исключения и пропуски;
- жёсткую дисциплину записи в память и в файловое зеркало;
- Slack-правила подтверждений сохранения;
- режим, в котором GitHub служит зеркалом, а не заменой рабочей среды агента.

## Статус
Это рабочий агент с настроенными инструкциями, навыком, файлами, Memory, Slack-каналом и GitHub-зеркалом. Карточка должна обновляться как часть обычного цикла сопровождения агента после значимых изменений.
