# Индекс текущих файлов агента

Назначение: этот файл фиксирует видимую часть текущих файлов агента для GitHub-зеркала и будущего воспроизведения похожих агентов.

Важно: индекс составлен по текущему видимому списку файлов в агенте. Поскольку файловое дерево отображается частично, этот список нужно дополнять по мере дальнейшей инвентаризации.

## 1. Служебные файлы развития агента

- `agent-development/github-mirror-manifest.md` — стартовый манифест структуры GitHub-зеркала, правил дублирования и подготовки будущих похожих агентов.

## 4. Что означает эта структура

- папка `protocols/` хранит предметные шаблоны, на которых держится логика сверки PDF-протоколов;
- папка `agent-development/` хранит служебные материалы для развития агента и зеркалирования в GitHub;
- папка `xl/` выглядит как технический набор файлов, связанных с табличными материалами и их внутренними ресурсами.

## 5. Что нужно добавить в GitHub-зеркало в первую очередь

В приоритетном порядке для зеркала:
1. все шаблоны из `protocols/`;
2. служебные материалы из `agent-development/`;
3. описание назначения папки `xl/` и решение, нужно ли зеркалить её целиком или только полезные артефакты.

## 6. Что ещё нужно дозаполнить позже

При следующей инвентаризации желательно дополнить:
- полный список файлов, если в агенте есть ещё элементы за пределами видимой части дерева;
- классификацию, какие файлы являются эталонными шаблонами, а какие техническими артефактами;
- отдельный индекс шаблонов по типам проверок и назначению.
