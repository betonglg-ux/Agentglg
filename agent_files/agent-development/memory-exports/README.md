# Экспорт Memory для GitHub-зеркала

Назначение: эта папка хранит переносимые выгрузки из Memory, которые важны для восстановления, клонирования и безопасной синхронизации похожего агента.

## Что складывать сюда
- подтверждённые паттерны ошибок;
- журнал пропущенных замечаний;
- заметки по шаблонам;
- подтверждённые поправки пользователей;
- устойчивые предпочтения пользователя по правилам проверки и формату ответов;
- журнал событий сохранения памяти;
- устойчивые правила, которые должны переживать будущие обновления и попадать в зеркало.

## Ключевой принцип
В Memory агент продолжает работать как обычно, и именно она остаётся первичным источником истины. В эту папку нужно выгружать только устойчивые и полезные для будущих агентов данные как файловый дубль для GitHub-зеркала.

## Обязательные правила
- запись в export-файл не заменяет запись в Memory;
- запись в Memory не заменяет export-дубль, если правило должно синхронизироваться с зеркалом;
- подтверждённое правило считается сохранённым только после проверки записи и в Memory, и в export-файле;
- события успешных и неуспешных сохранений тоже нужно фиксировать отдельным журналом;
- эти экспорты нужно поддерживать в актуальном состоянии как обычную рутинную часть сопровождения агента.

## Минимальный состав файлов
- `confirmed-error-patterns-export.md`
- `missed-findings-export.md`
- `template-notes-export.md`
- `user-corrections-export.md`
- `user-preferences-export.md`
- `memory-save-log-export.md`

## Как соотносить exports с рабочими файлами памяти
- `confirmed-error-patterns-export.md` <- `confirmed-error-patterns.md`
- `missed-findings-export.md` <- `missed-findings-log.md`
- `template-notes-export.md` <- `template-notes.md`
- `user-corrections-export.md` <- `user-confirmed-corrections.md`
- `user-preferences-export.md` <- `user-preferences.md`
- `memory-save-log-export.md` <- `memory-save-log.md`

## Когда обновлять эту папку
Обновляй её после каждого значимого изменения подтверждённых правил, паттернов, пропусков, заметок по шаблонам, пользовательских правок, пользовательских предпочтений или событий сохранения памяти, если эти данные должны попасть в GitHub-зеркало.

## Напоминание о направлении синхронизации
Синхронизация допустима только в сторону `текущий агент -> GitHub-зеркало`. Данные из зеркала не должны автоматически переопределять более новое локальное состояние агента.
