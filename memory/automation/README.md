# Автоматизация GitHub-зеркала

Эта папка подготавливает автоматический экспорт текущей рабочей среды в репозиторий `betonglg-ux/Agentglg` без ручной правки индексов и служебных файлов.

Главный принцип: локальная память агента в `/workspace/memory/memory` является источником истины, а GitHub-зеркало служит только внешним дубликатом и архивом. Экспорт не должен обновлять локальную память данными из зеркала.

Текущая версия автоматизации по умолчанию работает с веткой `main` репозитория `betonglg-ux/Agentglg` и при подготовке зеркала не должна наследовать случайный локальный `origin` от других служебных репозиториев.

Если запуск идет уже из локального checkout самого зеркала, автоматизация должна использовать его текущий настроенный `origin`, включая backend-remote вида `https://chatgpt.com/backend-api/git-authed/...`, а не принудительно переключаться на прямой адрес `github.com`.

## Что делает экспорт в зеркало

- копирует `AGENTS.md`, `agent_files/` и `memory/` в зеркало;
- перед каждым экспортом сохраняет локальный защитный снимок ключевых файлов памяти в `memory/snapshots/`;
- автоматически пересобирает `agent-development/`;
- заново генерирует индексы файлов и шаблонов;
- переносит ключевой навык `glavlab-protocol-review`;
- обновляет `CHANGELOG.md`;
- зеркалит все реальные `.md` файлы из памяти в `agent-development/memory-exports/raw-memory/`;
- останавливается, если в зеркале уже есть строки памяти, которых нет в локальной памяти;
- коммитит и, если доступ настроен, отправляет изменения в GitHub.
- не выполняет обратное восстановление памяти из GitHub-зеркала поверх локальных файлов.
- не подставляет устаревшие копии memory-файлов из `agent_files` или других служебных мест, если локальный файл памяти отсутствует.

## Один раз настроить доступ

```bash
export AGENTGLG_GITHUB_TOKEN=ваш_токен
bash /workspace/memory/memory/automation/setup_github_auth.sh
```

После этого токен будет сохранен в `/workspace/memory/memory/agentglg-github-token.txt`, и будущие экспорты смогут работать без ручной подстановки переменной окружения.

## Обычный запуск

```bash
bash /workspace/memory/memory/automation/export_agentglg_mirror.sh
```

## Запуск только при новых изменениях

```bash
bash /workspace/memory/memory/automation/export_agentglg_mirror.sh
```

В этом режиме скрипт сравнивает текущий отпечаток рабочей среды с последним успешным экспортом и ничего не отправляет, если новых изменений нет.

## Пробный запуск без push

```bash
bash /workspace/memory/memory/automation/export_agentglg_mirror.sh --no-push
```

## Защита от потери наработок

- если локальная память случайно стала беднее, чем файл в GitHub-зеркале, экспорт остановится с ошибкой вместо того, чтобы автоматически подтянуть данные обратно и смешать источники;
- перед каждой попыткой экспорта делается локальный защитный снимок ключевой памяти в `memory/snapshots/`;
- при таком стопе нужно сначала вручную сравнить локальную память, защитный снимок и подтвержденные правки пользователя; зеркало нельзя считать более правильным автоматически;
- если обязательный файл памяти отсутствует локально, экспорт тоже остановится, чтобы не выгрузить старую или подставную копию;
- принудительно обойти эту защиту можно только явным флагом:

```bash
bash /workspace/memory/memory/automation/export_agentglg_mirror.sh --allow-memory-overwrite
```

Использовать этот флаг стоит только тогда, когда вы уверены, что сознательно заменяете более старую версию памяти.
