# Подтверждённые правки и правила из Slack

## Статус

Рабочий постоянный файл памяти агента. Новые подтверждённые правила из Slack добавлять сюда после реальной записи в Memory и затем дублировать в экспортные файлы для зеркала, если они должны переживать будущие обновления.

- Что запрещено: нельзя подтверждать сохранение по пустому или неизменившемуся `slack-user-corrections.md`; нельзя считать проверкой простое чтение файла без сверки, что новая запись реально появилась.
- Обязательное действие после таких запросов: дописывать конкретную новую запись в `slack-user-corrections.md`, затем обновлять связанный рабочий memory-файл по смыслу и только потом обновлять export-файл для зеркала.

- Новое подтверждённое правило для протоколов КП(кубики): среднюю прочность определять по ГОСТ 10180-2012, пункт 8.4. По серии образцов использовать среднеарифметическое значение прочности испытанных образцов в серии: из двух образцов — по двум образцам; из трёх образцов — по двум образцам с наибольшей прочностью; из четырёх образцов — по трём образцам с наибольшей прочностью; из шести образцов — по четырём образцам с наибольшей прочностью.
- Новое подтверждённое правило для протоколов КП: разница между максимальным и минимальным значениями средней плотности образцов на момент проведения испытаний не должна превышать 50 кг/м³.
- Новое подтверждённое правило для всех без исключения проверок: если номер протокола указан в формате `/xx.yy`, где `xx` — месяц из даты протокола, а `yy` — год из даты протокола, это нужно отмечать как серьёзную ошибку.
- Как должно быть правильно: номер протокола должен иметь вид `№заявки`.`порядковый номер протокола`.`идентификационная аббревиатура`, далее пробел и `от ДАТА`.
- Дополнительное уточнение по текущему примеру: в данном случае ошибка присутствует в обоих протоколах.
- Корректировка правила по нескольким протоколам в одном файле: если обнаружены несколько разных протоколов (`№ X.1`, `X.2` и т.д.), а в нижнем колонтитуле стоит общее число страниц на весь пакет, это не считать ошибкой само по себе.
- Как действовать вместо этого: пользователю нужно давать напоминание `Обратите внимание, в файле сбита нумерация страниц и/или отсутствуют верхние колонтитулы. Печать протоколов производить из Excel файлов!`
- Область применения: это правило распространяется на все случаи, когда можно определить, что нумерация страниц в нижнем колонтитуле распространяется на несколько разных протоколов.
- Новое подтверждённое правило для протоколов КП: в протоколах КП нижний колонтитул по шаблону отсутствует, и это не является ошибкой.
- Новое подтверждённое правило для протоколов КП: в протоколах КП никогда не проверять нумерацию страниц и не выносить замечания по отсутствию, виду или последовательности страниц.
- Область применения: только протоколы КП; на другие типы протоколов это правило не распространяется.
