# Подтвержденные паттерны и исключения

- УК-протоколы: если у конструкции возраст 28 суток, а в выводе использована формулировка `более 28 суток`, это не считать ошибкой. По внутреннему правилу компании такой случай допустим и в замечания пользователю не выносить.
- УК-протоколы по схеме Г с приложением Б: обязательно сверять приложение с основным отчетом. Значения `Hi` из таблицы 1 приложения Б должны встречаться хотя бы один раз в отчете(ах), к которым относится приложение. Данные таблицы 2 приложения Б должны совпадать с данными отчета, включая наименование конструкции.
- УК-протоколы по схеме Г с приложением Б: если два разных отчета имеют одинаковое уравнение градуировочной зависимости, одинаковый коэффициент `b` и в основном листе есть примечание `Значения отрывов со скалыванием и определение коэффициента b представлены в приложении Б`, то наполнение приложения Б может быть одинаковым и может включать данные из обоих отчетов. При этом каждое приложение все равно обязательно относится к своему основному отчету по номеру. Если значения, уравнение или коэффициент `b` различаются, приложения Б должны отличаться по наполнению.
- КП-протоколы: отсутствие нижнего колонтитула не считать ошибкой. Для категории `№.КП` нижний колонтитул по форме не предусмотрен.
- Все отчеты и протоколы: номер проверять по формату `номер.подномер.идентификатор`. Если подномер отсутствует или серия не начинается с `1`, это ошибка нумерации.
- УК-протоколы: если в двух отчетах используется одинаковое уравнение градуировочной зависимости, но указан разный коэффициент `b`, это выносить в раздел замечаний или на уточнение, даже если это не подтвержденная ошибка.
- УК-протоколы: обязательно проверять орфографию в наименованиях конструкций. Явные опечатки внутри названия конструкции, например `пеекрытия` вместо `перекрытия`, выносить как отдельное замечание с точной локализацией по строке таблицы.
- Для всех будущих проверок PDF-протоколов: отдельно проверять орфографические и грамматические ошибки в видимом тексте документа, а не только расчеты, даты и оформление. Ошибки в наименованиях конструкций, заголовках таблиц, шапках и других текстовых блоках выносить как отдельные замечания с точной локализацией.
- УК-протоколы с приложениями-схемами: обязательно определять, к какому отчету относится каждое приложение, по номеру в нижнем колонтитуле. Если у отчета только одно приложение со схемой, то замечание по формулировке `Схема` / `Схемы` локализовать именно в `Приложении А`, а не в основном листе отчета.
- УК/УИ-протоколы: если в одной строке указана дата бетонирования диапазоном, например `15.03.2026 - 16.03.2026`, возраст проверять по крайней дате бетонирования. В таком примере `38 суток` считается корректным для даты `16.03.2026` и не является ошибкой.
- КУ-протоколы: вывод проверять по точному стилю формы, не придумывая свои названия. Меняется только число `обследуемого участка` / `обследуемых участков` по количеству оснований и тип основания из таблицы. Подтвержденные варианты:
  - одно основание, `песчаное основание`: `Коэффициент уплотнения обследуемого участка песчаного основания, на момент...`
  - более одного основания, `песчаное основание`: `Коэффициент уплотнения обследуемых участков песчаного основания, на момент...`
  - одно основание, `обратная засыпка`: `Коэффициент уплотнения обследуемого участка обратной засыпки, на момент...`
  - более одного основания, `обратная засыпка`: `Коэффициент уплотнения обследуемых участков обратной засыпки, на момент...`
  - одно основание, `грунтовое основание`: `Коэффициент уплотнения обследуемого участка грунтового основания, на момент...`
  - более одного основания, `грунтовое основание`: `Коэффициент уплотнения обследуемых участков грунтового основания, на момент...`
- Для всех PDF-протоколов: если в итоговом PDF сбит порядок страниц или отсутствуют верхние колонтитулы с логотипом и графикой, это не считать ошибкой протокола и не локализовать по страницам. Такой случай всегда выносить только отдельным предупреждением в самом конце отчета в точной формулировке: `Обратите внимание, в файле сбит порядок и отсутствуют верхние колонтитулы. Печать протоколов производить из Excel файлов!`
