# Подтвержденные паттерны ошибок

- 2026-04-28. Для УК-протоколов по схеме Г с приложением Б обязательно проверять, что значения `Hi` из таблицы 1 приложения Б присутствуют в отчете(ах), а данные таблицы 2 приложения Б идентичны данным основного отчета.
- 2026-04-28. Если в таблице 2 приложения Б указана конструкция, которой нет в основном отчете, это подтвержденная ошибка и ее нужно выносить в замечания.
- 2026-04-28. Орфографические ошибки в наименованиях конструкций считать реальными ошибками и выносить в замечания.
- 2026-04-28. Для УК-протоколов с приложениями-схемами нужно проверять принадлежность `Приложения А` к конкретному отчету по номеру в нижнем колонтитуле.
- 2026-04-28. Если у протокола только одно приложение со схемой, некорректное окончание `Схема` / `Схемы` в приложении считать замечанием.
- 2026-04-28. Если в отчетах одинаковое уравнение градуировочной зависимости, но разный коэффициент `b`, это не считать автоматической ошибкой, но обязательно выносить как замечание на внимание или уточнение.
- 2026-04-30. Для КУ-протоколов обязательно сверять вывод с таблицей по числу участков: если в таблице один участок, форма `обследуемых участков` в выводе является ошибкой.
- 2026-04-30. Для КУ-протоколов в выводе обязательно должен быть указан тип участка из таблицы, например `песчаное основание`, `обратная засыпка` или `грунтовое основание`.
- 2026-04-30. Для КУ-протоколов нельзя использовать произвольные формулировки вывода: нужно сверять с шаблонной фразой, где меняются только число участков и тип основания.
- 2026-04-30. Для отметок использовать такой формат: абсолютные отметки без знака, относительные со знаком `+` или `-`, кроме `0,000`.
- 2026-04-30. В отметках разделителем должна быть только запятая. Формат с точкой, например `140.200`, считать ошибкой.
- 2026-04-30. Для всех подходящих протоколов явные орфографические ошибки в словах, например `выполненых`, считать реальными замечаниями и отмечать отдельно.
- 2026-05-04. Для УК-протоколов с `Приложением А` все даты на схеме нужно отдельно сверять с датами бетонирования в основном протоколе. Если в легенде схемы есть дата, которой нет в таблице основного протокола, это является важным замечанием.
- 2026-05-04. Локальные разрывы рамок и границ таблиц нужно фиксировать даже тогда, когда остальная таблица выглядит нормально.
- 2026-05-05. Для всех без исключения проверок формат номера протокола вида `/xx.yy`, где `xx` — месяц из даты протокола, а `yy` — год из даты протокола, считать серьёзной ошибкой.
- 2026-05-05. Для всех без исключения проверок правильный формат номера протокола — `№заявки`.`порядковый номер протокола`.`идентификационная аббревиатура`, далее пробел и `от ДАТА`.
- 2026-05-05. Для протоколов КП(кубики) среднюю прочность определять по ГОСТ 10180-2012, пункт 8.4, по среднеарифметическому значению прочности испытанных образцов серии с отбором наибольших значений по правилу размера серии.
- 2026-05-05. Для протоколов КП разница между максимальным и минимальным значениями средней плотности образцов на момент проведения испытаний не должна превышать 50 кг/м³.
