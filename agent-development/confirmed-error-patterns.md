# Подтвержденные паттерны ошибок

Назначение: сюда нужно переносить из Memory подтверждённые пользователем типовые ошибки, которые агент должен особенно внимательно проверять в будущих запусках и в будущих похожих агентах.

## Обновлено: 2026-05-05

### Подтверждённые паттерны, которые нужно ловить

- Орфографические ошибки в наименованиях конструкций.
- Грамматические и текстовые ошибки в шапках, заголовках таблиц и выводах.
- Ошибки в номере протокола, если нарушен формат `номер.подномер.идентификатор`.
- Для всех проверок: если номер протокола указан в формате `/xx.yy`, где `xx` — месяц из даты протокола, а `yy` — год из даты протокола, это серьёзная ошибка.
- Для всех проверок: правильный формат номера протокола — `№заявки`.`порядковый номер протокола`.`идентификационная аббревиатура`, далее пробел и `от ДАТА`.
- Ошибки в выводах КУ-протоколов, если форма единственного/множественного числа не соответствует числу участков в таблице.
- Ошибки в выводах КУ-протоколов, если не указан тип участка из таблицы.
- Ошибки в выводах КУ-протоколов, если вывод сформулирован не по подтвержденной шаблонной фразе.
- Для УК-протоколов по схеме Г с приложением Б: отсутствие значений `Hi` из таблицы 1 приложения Б в отчете(ах).
- Для УК-протоколов по схеме Г с приложением Б: расхождение данных таблицы 2 приложения Б с данными отчета.
- Для УК-протоколов с приложениями-схемами: неверная принадлежность `Приложения А` к отчету, если номер в нижнем колонтитуле не совпадает.
- Для УК-протоколов с приложением-схемой: некорректное окончание `Схема` / `Схемы` в приложении, если приложение одно.
- Для УК-протоколов с `Приложением А`: наличие на схеме даты, которой нет в таблице основного протокола.
- Локальные разрывы рамок и границ таблиц, даже если остальная таблица визуально выглядит нормальной.
- Отдельные опечатки в словах типа `выполненых`.
- Для ВС-протоколов: не считать ошибкой отсутствие поля `Материал по проекту`; это допустимо и, при необходимости, выносится только как напоминание.
- Для ВС-протоколов: не считать ошибкой отсутствие ГОСТ в шапке.
- Для КП-протоколов: не считать ошибкой отсутствие нижнего колонтитула.
- Для КП(кубики): среднюю прочность определять по ГОСТ 10180-2012, пункт 8.4, по среднеарифметическому значению прочности испытанных образцов серии с отбором наибольших значений по правилу размера серии.
- Для КП: разница между максимальным и минимальным значениями средней плотности образцов на момент проведения испытаний не должна превышать 50 кг/м³.
- Для их протоколов: не считать ошибкой служебную нумерацию страниц в нижнем колонтитуле.
- Для их протоколов: не считать ошибкой формулировки `менее 28 суток` и `более 28 суток`, включая случай ровно `28` суток.
- Не считать дефектом протокола артефакты итоговой сборки PDF, где на лист может заезжать линия или фрагмент соседней страницы.

### Паттерны повышенного внимания

- Если в отчетах одинаковое уравнение градуировочной зависимости, но разный коэффициент `b`, это не автоматическая ошибка, но обязательное замечание уровня внимания или на уточнение.
- Для УК/УИ-протоколов, если дата бетонирования указана диапазоном, возраст нужно считать по крайней дате бетонирования.
- Для шаблона `2.0 ИПС универсальная` диапазоны с единицей измерения только после второго значения допустимы и не должны ошибочно попадать в замечания.

## Примечание

Этот файл хранит именно паттерны проверки: что считать ошибкой, что не считать ошибкой и что выносить как замечание уровня внимания.
