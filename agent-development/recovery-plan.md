# Пошаговый план восстановления агента из зеркала

Репозиторий зеркала: `betonglg-ux/Agentglg`

Назначение: этот документ описывает, в каком порядке восстанавливать текущего агента или собирать нового похожего агента на основе GitHub-зеркала.

## 1. Подготовить новый агент
1. Создать нового агента в редакторе.
2. Задать ему понятное имя, близкое к текущему назначению.
3. Убедиться, что агент будет работать в ChatGPT.
4. При необходимости сразу подключить GitHub, чтобы зеркало было доступно как внешний архив.

## 2. Восстановить инструкции
1. Открыть файл `agent-development/current-agent-instructions.md` из зеркала.
2. Скопировать его содержимое в системные инструкции нового агента.
3. Проверить, что в инструкциях сохранены:
   - роль агента;
   - ссылка на обязательный навык `glavlab-protocol-review`;
   - правила работы с шаблонами;
   - правила Memory;
   - режим GitHub Mirror.

## 3. Восстановить обязательный навык
1. Найти и прикрепить навык `glavlab-protocol-review`.
2. Свериться с `agent-development/skills/README.md`.
3. Проверить, что навык действительно доступен агенту и указан как основной регламент проверки.

## 4. Восстановить рабочие файлы
1. Открыть `agent-development/files-index/attached-files-index.md`.
2. Открыть `agent-development/files-index/templates-index.md`.
3. Восстановить папку `protocols/` и загрузить шаблоны по типам протоколов.
4. Если для работы нужны технические материалы из `xl/`, перенести и их после отдельной проверки необходимости.
5. Убедиться, что ключевые шаблоны действительно прикреплены к новому агенту.

## 5. Восстановить служебные файлы развития
1. Перенести в нового агента корневые файлы `AGENTS.md`, `automation/` и нужные файлы из `agent-development/`.
2. Как минимум восстановить:
   - `github-mirror-manifest.md`
   - `agent-summary.md`
   - `github-export-bundle.md`
   - `automation/README.md`
   - `automation/sync_agentglg_mirror.py`
   - `files-index/attached-files-index.md`
   - `files-index/templates-index.md`
   - `skills/README.md`
3. Проверить, что у нового агента есть читаемая база для дальнейшего сопровождения и зеркалирования.

## 6. Восстановить важные данные из Memory
1. Открыть четыре канонических файла памяти:
   - `agent-development/confirmed-error-patterns.md`
   - `agent-development/missed-findings-log.md`
   - `agent-development/template-notes.md`
   - `agent-development/user-confirmed-corrections.md`
2. При необходимости свериться с их прямыми копиями в `agent-development/memory-exports/raw-memory/`.
3. Перенести эти знания в рабочий контур нового агента.
4. Убедиться, что критичные наработки не потерялись и доступны новому агенту.

## 7. Восстановить GitHub-mirror режим
1. Убедиться, что новый агент подключён к GitHub.
2. Проверить, что в инструкциях сохранено правило: агент работает в своей среде, а GitHub хранит зеркало.
3. Проверить, что новый агент не использует GitHub как замену своей рабочей среды, а только как дублирующий архив.
4. При необходимости актуализировать путь и структуру зеркала в репозитории.

## 8. Проверить каналы
1. Убедиться, что ChatGPT-канал работает как основной.
2. Если нужен Slack, отдельно восстановить Slack-канал и его инструкции.
3. Проверить, что Slack не обязателен для базового восстановления, но обязателен для полной копии текущего агента.

## 9. Провести тест восстановления
1. Отправить новому агенту тестовый PDF-протокол.
2. Проверить, что он:
   - использует навык `glavlab-protocol-review`;
   - сверяет PDF с шаблонами;
   - возвращает структурированный список замечаний;
   - учитывает строгие правила локализации ошибок.
3. Если результат отличается от текущего агента, проверить:
   - полноту инструкций;
   - полноту шаблонов;
   - перенос навыка;
   - перенос накопленных знаний.

## 10. Закрепить восстановленную копию
1. После проверки обновить GitHub-зеркало, если при восстановлении структура была улучшена.
2. Зафиксировать отличия новой копии в `agent-development/CHANGELOG.md`.
3. Если новый агент стал рабочей веткой, продолжать дублировать его важные наработки в GitHub.

## Минимальный быстрый путь
Если нужно собрать похожего агента как можно быстрее, минимальный порядок такой:
1. Восстановить инструкции.
2. Прикрепить `glavlab-protocol-review`.
3. Загрузить папку `protocols/`.
4. Подтянуть четыре канонических файла памяти.
5. Проверить работу на одном тестовом PDF.

## Итог
Если выполнять восстановление именно в этом порядке, новый агент сохранит:
- основную логику проверки;
- обязательный навык;
- шаблоны;
- накопленные наработки;
- режим зеркалирования в GitHub.
