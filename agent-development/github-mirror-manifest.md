# GitHub Mirror Manifest

Репозиторий: `betonglg-ux/Agentglg`

Назначение: этот манифест описывает актуальную структуру GitHub-зеркала для текущего агента. Текущий агент продолжает работать со своими инструкциями, файлами, навыками и Memory, а GitHub хранит их файловые дубли, историю развития и материалы для быстрого создания будущих похожих агентов.

## Базовые принципы
- текущая конфигурация агента является основной рабочей средой;
- локальная Memory является первичным источником истины;
- GitHub является внешним зеркалом и архивом развития;
- важные наработки не должны оставаться только внутри Memory, если они нужны для переноса в другого агента;
- при расхождениях сначала нужно ориентироваться на актуальное состояние текущего агента, его локальной памяти, защитных снимков и подтверждённых пользовательских правок, а уже потом обновлять зеркало;
- синхронизация допустима только в сторону `текущий агент -> GitHub-зеркало`.

## Что должно храниться в разделах

### `agent-development/CHANGELOG.md`
Короткий журнал значимых изменений агента:
- новые правила проверки;
- важные исправления логики;
- изменения в навыках, памяти и Slack-правилах;
- обновления экспортов и структуры зеркала.

### `agent-development/current-agent-instructions.md`
Актуальная рабочая версия инструкций агента в переносимом виде.

### `agent-development/agent-summary.md`
Краткая карточка агента:
- назначение;
- каналы;
- источники истины;
- обязательные навыки и файлы;
- критичные правила памяти и зеркала.

### `agent-development/github-export-bundle.md`
Сводный список текущего состава полного зеркала и порядка его обновления.

### Симметричные зеркальные файлы рабочей памяти
Это одноимённые файловые дубли основных memory-файлов, которые должны обновляться синхронно с рабочей памятью агента:
- `agent-development/confirmed-error-patterns.md`
- `agent-development/missed-findings-log.md`
- `agent-development/template-notes.md`
- `agent-development/user-confirmed-corrections.md`
- `agent-development/user-preferences.md`
- `agent-development/slack-user-corrections.md`

Их назначение — не заменять Memory, а давать переносимую зеркальную копию для GitHub и восстановления похожего агента.

### `agent-development/memory-exports/`
Экспорт важных данных из Memory в переносимом виде.

Минимальные файлы:
- `README.md`
- `confirmed-error-patterns-export.md`
- `missed-findings-export.md`
- `template-notes-export.md`
- `user-corrections-export.md`

### `agent-development/skills/`
Зеркало материалов навыков.

Минимально:
- `README.md`
- отдельная папка на каждый важный навык;
- для текущего агента обязательно предусмотреть папку `glavlab-protocol-review/`.

### `agent-development/files-index/`
Индекс файлов агента, чтобы можно было быстро понять, какие материалы были прикреплены и зачем они нужны.

### `protocols/`
Основные шаблоны по типам испытаний.

### `xl/`
Технические табличные и вспомогательные ресурсы, если они нужны для восстановления проверочной среды.

## Что нужно фиксировать в GitHub в первую очередь
1. Актуальные инструкции агента.
2. Первичные зеркальные копии рабочих memory-файлов в `agent-development/`.
3. Экспортные дубли памяти в `agent-development/memory-exports/`.
4. Ключевой навык `glavlab-protocol-review`.
5. Карточку агента и bundle полного зеркала.
6. Индексы прикреплённых файлов и шаблонов.
7. Changelog изменений.

## Обязательный режим сопровождения зеркала
После каждого значимого изменения инструкций, памяти, навыков, экспортов памяти, правил Slack-подтверждений или структуры файлов агент обязан в том же рабочем цикле:
- определить, какие файловые дубли устарели;
- обновить их в `agent-development/` и `agent-development/memory-exports/`;
- проверить симметричные пары `Memory <-> agent-development/` для основных рабочих memory-файлов;
- добавить короткую датированную запись в `agent-development/CHANGELOG.md`.

Это не ручное исключение и не отдельная опция, а обычная обязательная часть сопровождения агента.

## Правила памяти и подтверждений
- подтверждённые правила должны сохраняться и в Memory, и в файловый экспорт для зеркала;
- сохранение не считается завершённым, пока не подтверждены обе записи;
- Memory остаётся первичной рабочей памятью, а зеркальные файлы в `agent-development/` и `memory-exports/` остаются дублями;
- для основных memory-файлов нужно поддерживать симметричную одноимённую структуру в `agent-development/`;
- в Slack для подтверждений памяти разрешены только жёсткие статусные шаблоны;
- свободные формулировки до реальной записи и проверки считаются ошибкой поведения агента.

## Что пригодится для размножения агента
При создании нового похожего агента нужно в первую очередь восстановить:
- инструкции из `current-agent-instructions.md`;
- структуру навыков из `skills/`;
- шаблоны из `protocols/`;
- устойчивые знания из Memory и её симметричных зеркальных копий;
- экспортные своды из `memory-exports/`;
- правила зеркала и changelog развития.

## Статус этого манифеста
Это рабочий актуальный манифест. Его нужно обновлять как часть обычного цикла зеркалирования, если меняются правила дублирования, симметрия памяти, структура экспортов или состав критичных файлов.
