# User Preferences

## 2026-04-28

- Если пользователь просит "дамб", собирать полноценный ZIP-архив всех файлов агента, которые реально доступны в текущей файловой среде, включая рабочие папки, память и локально доступные файлы навыков.
- После сборки дампа останавливаться на этом и не уходить в дополнительные промежуточные варианты, если пользователь отдельно не попросил иное.

## 2026-05-04

- В Slack и похожих диалогах перед сохранением правила в память сначала коротко писать пользователю о самом действии, например что правило сейчас сохраняется.
- После попытки сохранения всегда отдельной фразой явно отписывать итог: сохранено или не сохранено.
- Нельзя объединять обещание и факт в одну недоказанную формулировку вроде `запомнил`, если запись еще не проверена в постоянных файлах памяти.
- Для таких случаев использовать короткий стандартный шаблон из двух сообщений.
- Сообщение 1 перед действием: `Сейчас сохраню это правило в память агента и потом отдельно подтвержу результат.`
- Сообщение 2 после проверки успешной записи: `Правило сохранено в память агента.`
- Сообщение 2 при неуспешной записи: `Не удалось сохранить правило в память агента.`
